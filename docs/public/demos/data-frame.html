字段 大小 作用 说明
Header 2 bytes 帧同步 固定值 0xAA 0x55，用于识别帧开始
Length 1 byte 数据长度 命令 + 数据的总长度
Command 1 byte 命令码 标识帧类型和操作
Data 0~N bytes 数据区 实际传输的数据内容
CRC 2 bytes 校验码 CRC16 校验，检测传输错误
Tail 1 byte 帧结束 固定值 0x0D，可选字段

帧结构设计原则

设计要点

  • 帧头选择：使用特殊字节序列，避免与数据冲突
  • 长度字段：支持变长数据，提高灵活性
  • 命令码：按功能分类，预留扩展空间
  • CRC 校验：保证数据完整性，推荐 CRC16
  • 帧尾可选：调试阶段建议保留，发布可去掉

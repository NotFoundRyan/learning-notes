# 文章写作规范

> 本规范适用于技术笔记和博客文章的撰写，旨在保持文章风格统一、内容清晰易懂。

## 一、文档结构规范

### 1.1 文章头部（Front Matter）

```yaml
---
title: 文章标题 - 副标题（可选）
date: 2026-03-26
updated: 2026-04-01
tags: [标签1, 标签2]
category: embedded/data-structure
description: 简短描述文章内容
prerequisites:
  - /notes/c-language/pointers
  - /notes/data-structure/array
---
```

**字段说明：**

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | ✅ | 文章标题 |
| `date` | ✅ | 创建日期 |
| `updated` | ❌ | 最后更新日期 |
| `tags` | ✅ | 标签数组（不超过 5 个） |
| `category` | ❌ | 分类路径 |
| `description` | ✅ | 文章摘要 |
| `difficulty` | ❌ | 难度等级 |
| `prerequisites` | ❌ | 前置知识链接 |

### 1.2 标题层级

```markdown
# 一级标题（全文仅一个）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（谨慎使用）
```

**规则：** 标题层级不可跳级。

### 1.3 文章结构模板

**概念讲解类：** 定义 → 动机 → 原理 → 实现 → 陷阱 → 总结

**问题解决类：** 现象 → 排查 → 方案 → 举一反三

**对比分析类：** 背景 → 方案概述 → 详细对比 → 选型建议

### 1.4 要点速览

对于较长的文章，在开头提供快速摘要：

// 代码内容
```
````

### 3.2 代码块长度限制

- 单个代码块不超过 **40 行**
- 超过时拆分为多个代码块，中间穿插说明
- 展示完整代码时使用折叠组件：

```markdown
::: details 点击查看完整代码
```c
// 完整代码
```
:::
```

### 3.3 代码示例分级

| 级别 | 标记 | 含义 |
|------|------|------|
| 完整可运行 | `// 完整示例：可直接编译运行` | 包含 main 函数和头文件 |
| 代码片段 | `// 片段：需要补充上下文` | 只展示关键部分 |
| 伪代码 | `// 伪代码：展示思路` | 不保证语法正确 |

每个代码块的第一行注释应标明级别。

### 3.4 编译运行说明

对于完整可运行的代码，提供编译命令：

```markdown
**编译与运行：**
```bash
gcc -o array_list array_list.c -Wall -Wextra
./array_list
```

**预期输出：**
```
Array: [1, 2, 3, 4, 5]
Size: 5, Capacity: 8
```
```

### 3.5 代码说明规范

**核心原则：代码块下方必须添加详细的外部说明。**

| 内容 | 代码内部 | 代码外部 |
|------|---------|---------|
| "做了什么" | ❌ | ✅ 详细解释 |
| "为什么" | ✅ 简短注释 | ✅ 展开解释 |
| 关键标记 | ✅ `// [1]` | ✅ `[1]` 对应解释 |

#### 3.5.1 结构体说明

````markdown
```c
typedef struct {
    int *data;
    size_t size;
    size_t capacity;
} ArrayList;
```

**结构体成员说明：**

| 成员 | 类型 | 说明 |
|------|------|------|
| `data` | `int *` | 指向实际存储数据的内存块指针 |
| `size` | `size_t` | 当前数组中实际存储的元素个数 |
| `capacity` | `size_t` | 数组的总容量 |
````

#### 3.5.2 函数说明

````markdown
```c
ArrayList *array_list_create(size_t capacity)
{
    ArrayList *list = malloc(sizeof(ArrayList));
    if (list == NULL) return NULL;

    list->data = malloc(capacity * sizeof(int));
    if (list->data == NULL) {
        free(list);
        return NULL;
    }

    list->size = 0;
    list->capacity = capacity;
    return list;
}
```

**参数：** `capacity` - 初始容量

**返回值：** 成功返回指针，失败返回 `NULL`

**逐行解释：**
- `malloc(sizeof(ArrayList))` - 分配结构体内存（64 位系统约 24 字节）
- `malloc(capacity * sizeof(int))` - 分配数据内存（capacity=10 时为 40 字节）
- `free(list)` - 数据内存分配失败时释放结构体，避免泄漏
````

#### 3.5.3 关键逻辑标记

````markdown
```c
void *ptr = malloc(size);
if (!ptr) {
    errno = ENOMEM;    // [1] 设置错误码而非直接 exit
    return NULL;
}
```

[1] 设置 errno 而非 exit()，因为库函数不应替调用者决定错误处理策略。
````


### 3.6 小块代码 + 解释形式（推荐）

对于教学类文章，推荐使用"小块代码 + 解释"的形式，让读者逐步理解：

**核心原则：一个代码块只展示一个概念/函数/操作，紧跟解释。**

#### 3.6.1 示例格式

````markdown
```python
df = pd.read_csv('data.csv')
```

**解释：**
- `pd.read_csv()` 读取 CSV 文件，返回 DataFrame 对象
- DataFrame 是 pandas 的核心数据结构，类似 Excel 表格

---

```python
X = df.drop('churn', axis=1).values
y = df['churn'].values
```

**解释：**
- `df.drop('churn', axis=1)` 删除标签列，`axis=1` 表示按列操作
- `.values` 将 DataFrame 转换为 numpy 数组
````

#### 3.6.2 格式要点

| 要点 | 说明 |
|------|------|
| 代码块大小 | 每个代码块 1-10 行，展示一个概念 |
| 分隔符 | 代码块之间用 `---` 分隔 |
| 解释位置 | 解释紧跟在代码块下方 |
| 解释格式 | 使用 `**解释：**` 开头，然后逐条说明 |
| 数据变化 | 对于数据处理，添加数据变化图示 |

#### 3.6.3 数据变化图示

对于数据处理类代码，添加数据变化图示：

```
原始 CSV                    特征 X                     numpy 数组
┌─────┬───────┬───────┬───────┐    ┌─────┬───────┬───────┐    shape: (10000, 14)
│ age │ ...   │ churn │       │    │ age │ ...   │       │
├─────┼───────┼───────┤   →   ├─────┼───────┤   →
│  25 │ ...   │   0   │       │    │  25 │ ...   │
└─────┴───────┴───────┘       └─────┴───────┘
```

#### 3.6.4 适用场景

| 场景 | 推荐形式 |
|------|---------|
| 教学/入门文章 | ✅ 小块代码 + 解释 |
| API 文档 | 完整代码 + 参数表格 |
| 进阶/实战文章 | 渐进式展示 |
| 快速参考/速查 | 代码片段 + 简短注释 |

## 四、语言风格规范

### 4.1 中英文混排

- 中文为主，技术术语可使用英文
- **中英文之间需要有空格**，**中文与数字之间需要有空格**

```markdown
✅ 状态机（State Machine）是一种行为模型。环形缓冲区的大小通常是 256 字节。
❌ 状态机(State Machine)是一种行为模型。环形缓冲区的大小通常是256字节。
```

### 4.2 标点符号

- 中文句子使用中文标点（，。！？）
- 英文句子使用英文标点（,.!?）
- 代码中的标点使用英文标点

### 4.3 写作语气

| 场景 | 人称 | 示例 |
|------|------|------|
| 解释概念 | "我们" | ✅ "我们先来看一下内存布局" |
| 指导操作 | 第二人称或无主语 | ✅ "将头文件包含到项目中" |

**语气定位：** 像耐心的同事在白板前讲解，不居高临下，不说"显然"、"很简单"。

### 4.4 术语一致性

| 统一用词 | 避免混用 |
|---------|---------|
| 环形缓冲区 | 循环缓冲区、圆形缓冲区、Ring Buffer |
| 时间复杂度 | 时间开销、运行效率 |

首次出现时标注英文：环形缓冲区（Ring Buffer），后续直接使用中文。

## 十一、特殊内容处理

### 11.1 错误信息展示

```markdown
**错误现象：**
```
Segmentation fault (core dumped)
```

**错误代码：**
```c
int *ptr = NULL;
*ptr = 42;  // ← 错误在这里
```

**原因分析：** 对 NULL 指针解引用触发段错误。

**修复方案：**
```c
int *ptr = malloc(sizeof(int));
if (ptr != NULL) {
    *ptr = 42;
}
```
```
